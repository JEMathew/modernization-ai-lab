# Enterprise Context Graph

## What It Is

The Enterprise Context Graph is the knowledge layer that connects all modernization inputs, decisions and outputs.

It is designed to make modernization workflows context-aware, reusable and auditable.

## What It Stores

- Source platforms
- Schemas, tables and columns
- SQL and stored procedures
- ETL jobs and BI reports
- Data lineage
- Dependencies
- Business rules
- Data quality checks
- Source-to-target mappings
- Human review decisions
- Validation evidence
- Cutover decisions
- Exceptions and lessons learned

## Why It Matters

Without a shared context graph, AI agents behave like isolated tools. With a context graph, each migration improves the next one.

## Moat

The Enterprise Context Graph becomes a proprietary modernization knowledge base that compounds over time through:

- Customer-specific context
- Reusable migration patterns
- Human approvals
- Validation outcomes
- Model evaluation feedback
- Enterprise governance decisions
